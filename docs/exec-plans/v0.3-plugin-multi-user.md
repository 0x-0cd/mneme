# V0.3 — Plugin System + Multi-User Support

## 1. 竞品调研结论

### 插件系统

| 方案 | 做法 | 复杂度 | 适合 Mneme? |
|:----|:----|:-----:|:----------:|
| **Mem0** | Python 装饰器注册 + 事件钩子，pip 安装即插 | 低 | ✅ |
| **LangChain** | BaseCallbackHandler 接口，每个事件调用 | 中 | ✅ |
| **Zep** | 无插件架构，只暴露 API | — | ❌ |
| **FastAPI-style** | APIRouter 挂载，运行时可热插拔 | 中 | ✅ |
| **Sqlite-vec** | 编译扩展加载 `.so` | 高 | ❌ |

### 多用户

| 方案 | 隔离方式 | 共享存储 | 运营成本 |
|:----|:--------|:--------:|:-------:|
| **Mem0** | 每条记忆带 `user_id` | 单 DB，行级隔离 | 低 |
| **Zep** | session/user 前缀 | 单 DB，前缀隔离 | 低 |
| **Letta** | 每个 agent 独立 DB | 独立 SQLite 文件 | 中 |
| **Memobase** | 用户级命名空间 | MongoDB 集合隔离 | 中 |
| **FastGPT** | user_id 列 + 权限 | 单 DB | 低 |

**结论：** Mneme 走 **行级 user_id 隔离** + **装饰器注册事件钩子**

### 3. 多用户设计

```
用户 A → user_id="user_a" → 所有记忆带 user_id 字段
用户 B → user_id="user_b" → 同上，同 DB

API:
  POST /v1/memories { user_id: "user_a", ... }
  GET /v1/memories?user_id=user_a
  GET /v1/users/{user_id}/stats

隔离级别：行级（user_id 列 + 索引）
扩展：可选改为独立 DB 文件（未来）
```

## 4. 分步实现

### Step 1: Plugin 框架核心

`src/mneme/plugin/` — 基础架构

- `event.py` — Event 数据类 + EventType 枚举
- `bus.py` — EventBus（同步通知 + 异常隔离）
- `base.py` — PluginBase 抽象基类
- `registry.py` — PluginRegistry（加载/注册/卸载）

### Step 2: 示例插件

- `PluginLogger` — 事件发生时打印日志
- `PluginWebhook` — 事件时 POST 到指定 URL

### Step 3: 多用户支持

- `user_id` 字段加入 Memory 模型
- 所有接口增加 `user_id` 参数
- 搜索/统计等过滤逻辑加 `user_id` 条件
- 索引、迁移脚本

### Step 4: CLI + API

- `mneme plugin list / load / unload`
- `GET /v1/users/{user_id}/memories`

### Step 5: 自测

- 16+ 个测试覆盖插件注册/事件/多用户隔离
- 模拟 pip 安装式插件加载（开发期用 PYTHONPATH）
