# V0.2a — 矛盾检测（Conflict Detection）

> 目标：检测记忆库中语义矛盾的内容，提升记忆质量。
> 对应设计文档中的 `Quality Module`。

## 数据模型

```python
@dataclass
class Contradiction:
    memory_a_id: str
    memory_b_id: str
    content_a: str
    content_b: str
    type: ContradictionType  # 矛盾类型
    similarity: float        # 语义相似度
    score: float             # 矛盾可信度 [0, 1]
    reason: str              # 可读解释

class ContradictionType(StrEnum):
    DIRECT = "direct"        # 直接对立（"喜欢" vs "讨厌"）
    TEMPORAL = "temporal"    # 时间矛盾（"在杭州"→"在上海"，无时间标记）
    VALUE = "value"          # 值矛盾（权重/偏好差异）
    PARTIAL = "partial"      # 部分矛盾（限定条件不一致）
```

## 检测策略

### 1. 语义搜索（找候选对）
- 用现有 `Searcher.search()` 找语义相似的记忆对
- 阈值：相似度 > 0.7 的视为"相关"，需要进一步判断

### 2. 矛盾判断（两种模式）
| 模式 | 方法 | 依赖 |
|:----|:----|:----:|
| **基础模式**（默认） | 基于嵌入 + 内容启发式判断 | 无外部依赖 |
| **LLM 模式**（可选） | 用 LLM Judge 判断矛盾 | 需 LLM API |

### 3. 基础模式启发式规则
- 同类型 + 高语义相似度 + 情感极性相反（简单情感词匹配）
- 同类型 + 相同标签 + 同一属性的不同值（规则匹配）
- 同一实体在不同时间点的状态冲突
