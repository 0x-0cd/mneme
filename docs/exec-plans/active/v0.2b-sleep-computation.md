# V0.2b 睡眠计算 — 执行计划

> 记忆整理：合并相似、衰减遗忘、统计报告

---

## 1. 设计思路

睡眠计算模拟生物体在睡眠时的大脑记忆整理过程（记忆巩固 + 突触修剪），
在 Mneme 中实现为可手动触发的后台维护流程。

### 核心操作

```
睡眠周期 ─→ 合并相似记忆（consolidate）
           → 时间衰减权重（decay）
           → 生成报告（report）
```

### 合并策略（Consolidate）

1. 遍历所有活跃记忆
2. 用 embedding 余弦相似度找近邻（threshold ≥ 0.92）
3. 同类型 + 高相似 → 合并：
   - **内容**：取较长/较新的
   - **标签**：union
   - **权重**：两者之和（max capped at 2.0）
   - **metadata**：merge（后者覆盖前者）
   - 旧记忆标记 `superseded_by` → 新记忆 ID
   - 旧记忆软删除

### 衰减策略（Decay）

```
weight_new = weight_old * 0.5 ^ (days_since_update / halflife_days)
```

- `halflife_days` 默认 30 天
- `forget_threshold` 默认 0.05
- 低于 threshold 的记忆打标记（不自动删除）
- weight 更新到 DB

## 4. 模块设计

### SleepEngine

```python
class SleepReport:
    consolidated: int       # 合并了多少对
    decayed: int            # 衰减了多少条
    forgotten: int          # 低于遗忘阈值的条数
    total_before: int       # 处理前总数
    total_after: int        # 处理后总数
    duration_ms: int        # 耗时

class SleepEngine:
    def __init__(self, db, vindex, embed, searcher):
        ...

    def run_cycle(self, dry_run=False) -> SleepReport:
        """执行完整睡眠周期"""
        total_before = self.db.count()
        t0 = time.time()
        consolidated = self.consolidate(dry_run)
        decayed = self.decay(dry_run)
        total_after = self.db.count()
        return SleepReport(...)

    def consolidate(self, threshold=0.92, dry_run=False) -> int:
        """合并相似记忆"""

    def decay(self, halflife_days=30, forget_threshold=0.05, dry_run=False) -> tuple[int, int]:
        """衰减权重 + 标记遗忘"""
```

### CLI

```bash
mneme sleep [--dry-run]
```

输出示例：
```
💤 睡眠周期完成
  ├─ 合并相似记忆: 3 对
  ├─ 权重衰减: 12 条
  ├─ 建议遗忘: 2 条
  └─ 耗时: 156ms
```

### API

```
POST /v1/sleep?dry_run=true
→ { "consolidated": 3, "decayed": 12, "forgotten": 2, ... }
```

## 6. 约束

- consolidation 的阈值和 decay 参数可配置（构造函数参数）
- 不自动触发，必须手动调用（V2 再考虑 cron/定时）
- 不影响现有 CRUD / 搜索功能
- 测试使用 FakeEmbeddingModel（同其他模块）
