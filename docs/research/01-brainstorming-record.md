# 脑暴记录 — AI 记忆系统

> 日期：2026-06-13
> 参与者：Architect + Emma (PM)

## 定位共识

> 做一个**纯离线、即插即用**的 AI Agent 记忆系统，主动适配各种 Agent，留出强扩展性接口。

## 差异化三板斧

### ① 端侧优先（Edge-First）

不能跑在本地的记忆系统都是伪需求。**离线可用，在线可选**。

- Raspberry Pi 上 `pip install` 就能跑
- 零外部依赖（不用 Docker、不用 Neo4j、不用云 API Key）
- 对标 Mem0 的"3行代码接入"但本地就能跑满血版

### ② 记忆质量管理

大多数记忆系统只管"存没存"，不管"对不对"。我们要做：

- **矛盾检测** — 用户今天说"爱吃辣"，昨天说"胃不好"，系统能发现矛盾
- **过期管理** — 3 年前的偏好自动降权
- **修正学习** — 用户说"不对，我现在变了"，系统记得纠正

### ③ 留出扩展接口

- HTTP API 封装（localhost 轻量服务）
- MCP 协议优先
- 插件生命周期（Obsidian 式）

## 记忆分层设计

参考脑科学 + MemPalace 四层结构：

```
本能/潜意识（系统身份）→ 即时记忆（当前对话）
    → 短期/热记忆（近7天）→ 长期/冷记忆（归档）
```

### 类型化记忆

不同记忆类型走不同存储和检索策略：

| 类型 | 示例 | 存储策略 | 检索策略 |
|:----|:-----|:--------|:--------|
| 事实记忆 | "用户叫张三，Python 开发者" | KV 结构化 | 精确匹配 |
| 偏好记忆 | "用户喜欢暗色模式，用 Vim" | 向量搜索 | 语义匹配 |
| 事件记忆 | "上个月部署了博客" | 时间线 | 时序检索 |
| 对话记忆 | "刚才聊到记忆系统" | 短期缓存 | 自动过期 |
| 技能记忆 | "项目用了 FastAPI + SQLAlchemy" | 标签索引 | 分类匹配 |

### 权重衰减机制

参考艾宾浩斯遗忘曲线：`R = e^(-t/S)`

- 新记忆写入 → 初始权重 1.0
- 每次被检索/引用 → 权重 +0.1（强化）
- 随时间按艾宾浩斯曲线衰减
- 权重低于阈值 → 自动归档到冷存储
- 权重接近 0 → 可被自动清理

## 睡眠模式（AI 做梦）

系统空闲时自动运行的后台整理过程：

1. 权重衰减计算
2. 矛盾检测（发现"爱吃辣"vs"胃不好"）
3. 冷热记忆分层整理
4. 预计算常见查询的响应缓存

参考实现：Letta Sleep-time Compute（双 Agent 架构）、Claude Code Auto-dream（24h 触发）

### Sleep-time 关键数据

| 指标 | 改善 |
|:----|:----:|
| 测试时计算量 | 降低 **5x**（同等准确度） |
| 每次查询平均成本 | 降低 **2.5x** |
| 长期记忆准确率 | 提升 **18%** |

> 数据库类比：睡眠 = PostgreSQL 的 VACUUM + 物化视图刷新

## 轻量版本控制

每条记忆记录扩展字段：

```
- content: 当前内容
- created_at: 创建时间
- updated_at: 最后修改时间
- version: 版本号（从 1 递增）
- superseded_by: 被哪条新记忆替代（null=当前有效）
```

MVP 阶段不搞全量 git（blame/diff/branch），但留接口给未来。

## 隐私策略

- 单条记忆删除
- 全部记忆清除（factory reset）
- 数据文件在哪，用户自己管理
- **提供彻底清除能力，用不用由用户决定**

## 存储引擎选择

SQLite + sqlite-vec 向量扩展

| 指标 | 数据 |
|:----|:----:|
| p99 延迟 (384维, 10万向量) | **4ms** |
| 内存占用 | 45MB |
| 包体大小 | 800KB |
| 索引构建 | 无需（暴力扫描优化到极致）|
| 召回率（int8） | 99.8% |

10 万向量以下、384 维的典型场景，sqlite-vec 完全够用。

## 设计原则

- **存储原文，不做盲目摘要**（参考 MemPalace 教训：提取必然丢失信息）
- **但提取结构化元数据做索引**，兼顾全文检索和快速命中
- **存原文 + 元数据双通道**
