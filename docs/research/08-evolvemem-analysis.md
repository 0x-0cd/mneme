# EvolveMem: Self-Evolving Memory Architecture via AutoResearch

> **标题:** EvolveMem: Self-Evolving Memory Architecture via AutoResearch for LLM Agents
> **来源:** arXiv:2605.13941 (2026-05-13)
> **代码:** https://github.com/aiming-lab/SimpleMem

---

## 一、核心论点

> "现有记忆系统只让存储内容演变，但检索基础设施在部署后完全冻结。"

EvolveMem 认为真正的自适应记忆需要**两层共进化**：
1. **存储的知识**（内容、合并）
2. **检索机制**（评分函数、融合策略、答案生成策略）

---

## 二、架构：三层 + 自进化闭环

### 2.1 结构化记忆存储

- **6 类记忆：** episodic / semantic / preference / project state / summary / procedural
- **提取：** 滑动窗口 + 重试 + 块拆分 + 覆盖验证
- **合并策略：**
  - 去重（Jaccard τ=0.80）
  - **重要性衰减**：线性 0.05/天，下限 0.15
  - **实体强化**：δ=0.05/次共现，上限 0.30

### 2.2 可进化检索层（3 种视图）

| 视图 | 方法 |
|:----|:----|
| Lexical | BM25 精确关键字匹配 |
| Semantic | BGE-base-en-v1.5 稠密嵌入余弦相似度 |
| Structured | 元数据过滤（实体、人物、位置） |

**融合模式：** `sum` / `weighted-sum` / `rrf`（倒数排名融合）

**评分函数：**
```
s(q, m_i; θ) = s_fuse(q, m_i; θ) + λ_ι * ι_i + λ_r * rec(m_i) + ρ_i
```

**查询增强：**
- 对抗性实体交换：剥离误导人名，按主题重新搜索
- 查询分解：LLM 将多跳问题分解为子查询

**答案生成：** 每类记忆可配置（简洁/解释性/验证性/推断性），可选第二遍验证

### 2.3 自进化引擎（AutoResearch）

**核心思想：用 LLM 诊断自身失败**
1. 读取 per-question 失败日志
2. 分类根因：错误实体 / 上下文不足 / 时间混淆
3. 提出针对性配置调整 Δθ

**守卫机制（三段式）：**
```
θ_{r+1} = {
    θ*_{r-1}            if f_{r-1} - f_r > τ_rev         (回退)
    θ_r ⊕ η_exp          if 停滞2轮                      (探索)
    clamp_Θ(θ_r ⊕ Δθ_r)  otherwise                       (应用)
}
```

**收敛条件：** f_r - f_{r-1} < 0.005 或 最多 R_max=7 轮

**Figure 1 进化轨迹：** GPT-4o 从 30.5% 基线 → 54.3% 在 R7。R2 有一个 harmful proposal 被自动回滚，全程无人干预。

## 三、实验结果

### 3.1 LoCoMo 主结果（GPT-4o, F1分数）

| 方法 | F1 | 相对提升 |
|:----|:--:|:--------:|
| MemVerse | 0.365 | — |
| Mem0 | 0.397 | 基准 |
| MemGPT | 0.404 | +1.8% |
| SimpleMem | 0.432 | +8.8% |
| **EvolveMem** | **0.543** | **+36.8% vs Mem0** |
| 未进化基线 | 0.305 | 初始状态 |

> "最大提升在 temporal (+63.4%) 和 single-hop (+68.7%) 类别"

### 3.2 MemBench 主结果（GPT-4o, 准确率）

| 方法 | 准确率 |
|:----|:----:|
| RecentMem | 57.1% |
| MemGPT | 57.1% |
| **EvolveMem** | **67.9%** |

> "超过最强基线 18.9% 相对提升"

### 3.3 跨基准迁移（重要发现 ✅）

| 配置 | LoCoMo F1 | MemBench Acc |
|:----|:-------:|:----------:|
| 𝒞_L（仅 LoCoMo 进化） | **0.543** | 67.0% |
| 𝒞_LM（LoCoMo → 继续进化 MemBench） | 0.541 | **67.9%** |
| 𝒞_M（仅 MemBench 进化） | 0.527 | 67.5% |

> 在 LoCoMo 学到的配置迁移到 MemBench 时**性能保持而非下降**，说明学到了通用检索原则而非基准特定技巧。

### 3.4 消融实验

| 移除组件 | F1 变化 |
|:--------|:-------:|
| 自进化（冻结初始 θ） | **-23.8 pp**（最大） |
| 衰减/强化机制 | -9.6 pp |
| 第二遍答案验证 | -8.0 pp |
| 元数据过滤 | -5.8 pp |

## 四、与 Mneme 的对比

| 维度 | EvolveMem | Mneme |
|:----|:---------|:------|
| **记忆分层** | 6 类 + 线性衰减 0.05/天 | 艾宾浩斯指数衰减（理论优越） |
| **自进化** | ✅ LLM 驱动的 AutoResearch（重） | ❌ 计划中（睡眠周期，轻量） |
| **检索视图** | BM25 + Dense + 元数据 | 向量 + BM25 |
| **融合策略** | sum/weighted-sum/rrf | 语义分数（待改进） |
| **LoCoMo F1** | **0.543**（GPT-4o, 7轮自进化） | 全量跑中 |
| **部署成本** | GPT-4o 云端（昂贵） | DeepSeek V4 Flash 边缘优先 |
| **自进化成本** | 每轮完整跑一次基准（昂贵） | 计划抽样诊断（轻量） |

---

## 五、对 Mneme 的启发

1. **0.543 是天花板不是地板** — 用 GPT-4o + 7 轮跑基准才达到。轻量方式接近就赢了
2. **"诊断-建议-守卫"自进化循环** — 与我们睡眠周期设计一致但太重，需做边缘友好版
3. **线性衰减 0.05/天** — 比我们的指数衰减粗糙，我们在遗忘曲线上有理论优势
4. **跨基准正迁移** — 验证自进化不会过拟合，对我们设计睡眠周期是好信号
5. **消融：自进化贡献最大（-23.8pp）** — 说明进化机制本身比具体配置更重要
